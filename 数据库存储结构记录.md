## 主要业务数据库 (esg)

### 1. dids 表 - 用户身份信息表
**表位置**: `esg.dids`  
**用途**: 存储用户注册信息、DID身份、角色权限等

#### 表结构
```sql
DESCRIBE esg.dids;
+-------------------+-----------------+------+-----+-------------------+-----------------------------------------------+
| Field             | Type            | Null | Key | Default           | Extra                                         |
+-------------------+-----------------+------+-----+-------------------+-----------------------------------------------+
| id                | bigint unsigned | NO   | PRI | NULL              | auto_increment                                |
| name              | varchar(64)     | YES  | MUL | NULL              |                                               |
| phone             | varchar(32)     | YES  |     | NULL              |                                               |
| email             | varchar(128)    | YES  | MUL | NULL              | 邮箱地址（唯一标识）                           |
| password          | varchar(128)    | YES  |     | NULL              | 密码哈希                                       |
| role              | varchar(32)     | YES  | MUL | NULL              | 用户角色（owner/property_manager/institution） |
| age               | bigint          | YES  |     | NULL              | 用户年龄                                       |
| created_at        | varchar(32)     | YES  |     | NULL              | 创建时间                                       |
| did               | varchar(128)    | YES  | UNI | NULL              | DID标识符（唯一）                              |
| created_timestamp | timestamp       | YES  |     | CURRENT_TIMESTAMP | DEFAULT_GENERATED                             |
| updated_timestamp | timestamp       | YES  |     | CURRENT_TIMESTAMP | DEFAULT_GENERATED on update CURRENT_TIMESTAMP |
| full_name         | varchar(64)     | YES  |     | NULL              | 用户全名                                       |
| building_name     | varchar(128)    | YES  |     | NULL              | 建筑名称（Owner角色专用）                      |
| building_addr     | varchar(256)    | YES  |     | NULL              | 建筑地址（Owner角色专用）                      |
| building_type     | varchar(64)     | YES  |     | NULL              | 建筑类型（Owner角色专用）                      |
| property_name     | varchar(128)    | YES  |     | NULL              | 物业名称（Property Manager角色专用）           |
| occupation        | varchar(64)     | YES  |     | NULL              | 职业信息                                       |
| institution       | varchar(128)    | YES  |     | NULL              | 机构名称（Institution角色专用）                |
| status            | varchar(32)     | YES  |     | pending           | 用户状态（pending/verified/completed）         |
+-------------------+-----------------+------+-----+-------------------+-----------------------------------------------+
```

#### 索引信息
- **主键**: `id` (auto_increment)
- **唯一索引**: `did` (DID标识符)
- **普通索引**: `email`, `role`, `name`, `building_name`, `institution`, `status`

### 2. esg_files 表 - ESG文件存储表
**表位置**: `esg.esg_files`  
**用途**: 存储ESG相关文件的上传记录、元数据、IPFS哈希等

### 3. fabric_assets 表 - 区块链资产表
**表位置**: `esg.fabric_assets`  
**用途**: 存储Hyperledger Fabric链码中的资产信息

## 数据存储策略

### 1. 用户注册数据流
```
前端注册 → 后端验证 → 存储到 esg.dids 表 → 同步到 fabric-sdk.dids 表
```

### 2. 文件存储数据流
```
文件上传 → IPFS存储 → 元数据存储到 esg.esg_files 表 → 区块链记录
```

### 3. 区块链数据流
```
链码操作 → 数据存储到 fabric_assets 表 → 同步到区块链网络
```

## 数据库操作命令

### 常用查询命令
```bash
# 连接数据库
docker exec -it fabric-sdk-mysql mysql -uroot -pTest@123456

# 查看数据库
SHOW DATABASES;

# 使用数据库
USE esg;
USE `fabric-sdk`;

# 查看表
SHOW TABLES;

# 查看表结构
DESCRIBE table_name;

# 查看数据
SELECT * FROM table_name LIMIT 10;

# 统计记录数
SELECT COUNT(*) FROM table_name;
```

### 数据清理命令
```bash
# 删除所有用户记录
DELETE FROM esg.dids;

# 删除特定邮箱记录
DELETE FROM esg.dids WHERE email='example@email.com';

# 重置自增ID
ALTER TABLE esg.dids AUTO_INCREMENT = 1;
```

## 注意事项

### 1. 数据一致性
- 两个数据库的 `dids` 表结构略有不同
- 主要业务数据存储在 `esg` 数据库
- SDK配置数据存储在 `fabric-sdk` 数据库

### 2. 备份建议
- 定期备份 `esg` 数据库（业务数据）
- 备份前确认数据完整性
- 建议使用 `mysqldump` 进行备份

### 3. 权限管理
- 生产环境建议创建专用数据库用户
- 限制数据库访问权限
- 定期更新数据库密码
