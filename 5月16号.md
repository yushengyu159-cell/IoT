### 目标

### 后端 API 接口
- **查询**: `GET /api/multichannel/query?channel=xxx&fn=xxx&args=[...]` → 返回 `{"code": 0, "data": ...}`

- **通道列表**: `GET /api/multichannel/channels` → 返回 `{"code": 0, "channels": [...]}`

### 实现方式

- `mcApi.query(channel, fn, args)` - 查询区块链
- `mcApi.invoke(channel, fn, args)` - 调用区块链
- `mcApi.now()` / `mcApi.sd()` - 时间工具

#### Step 2: 在4个组件中追加区块链 HTML 区域

| 组件 | 追加内容 | 使用通道 |
|------|----------|----------|
| **DeviceMgt** | 门禁权限管理区域（刷新/授权/测试按钮） | access-channel |
| **DataVault** | ESG 链上文件区域（刷新/上传/提交报告） | esg-channel |
| **Maintenance** | 维护工单区域（刷新/创建/测试） | maintain-channel |
| **Finance** | 账单缴费区域（刷新/创建账单/抄表/测试） | billing-channel |

每个区域包含：
- 标题（显示通道名称）
- 操作按钮（Refresh / Create / Test）
- 数据展示区（`<div id="xxxChainData">`）

#### Step 3: 插入 Vanilla JS 区块链交互逻辑
在 `ReactDOM.createRoot` 之前插入 IIFE，为每个通道实现：
- 刷新按钮：查询链上数据，渲染统计卡片 + 数据表格
- 创建/测试按钮：弹出 prompt 输入参数，调用 invoke 写入链上

## 三、i18n 三语切换尝试（失败，已回退）

### 目标
实现前端 EN/简/繁 三语切换功能。页面右上角已有语言切换按钮（EN/简/繁），但大部分组件的文本是硬编码中文/英文，未使用 `t()` 翻译函数。

### 技术架构
- React 18 + Babel 单文件 SPA（`esg-dashboard.html`）
- i18n 系统: `I18nContext` + `useI18n()` + `t('key')` 模式
- 已有3个语言块: `en`（182 keys）、`zh-CN`（182 keys）、`zh-TW`（182 keys）
- 语言切换按钮位于 Header 组件右上角

### 问题分析
通过 `check_i18n2.py` 分析发现：
- 79个翻译 key 已定义但未在组件中通过 `t()` 使用
- 组件中大量硬编码中文文本（Dashboard ~50处，Sustainability ~80处，Finance ~30处，AI Agent Panel ~20处）
- `vs` key 在 zh-CN 和 zh-TW 中缺失

### 尝试过程

#### 尝试1: 正则批量替换（fix_i18n.py）— 白屏
- 使用 `re.sub()` 批量替换195处硬编码文本为 `t()` 调用
- **结果**: Babel 编译失败，页面白屏
- **原因**: 正则替换破坏了 JSX 语法结构（引号不匹配、花括号嵌套错误等）
- **处理**: 从 `.bak` 备份恢复

#### 尝试2: 精确字符串替换（safe_i18n.py）— 白屏
- 改用 `str.replace()` 精确匹配替换203处文本
- 分两步执行：
  1. `add_i18n_keys.py`: 向3个语言块各添加约150个新翻译 key（en/zh-CN/zh-TW）
  2. `safe_i18n.py`: 逐个替换硬编码文本为 `t()` 调用
- **结果**: 203处替换成功，19处未匹配，但页面再次白屏
- **原因**: 替换范围过大，触碰了 JSX 属性值（如 `placeholder`）和字符串内的内容，导致 Babel 编译失败
- **处理**: 从 `.bak3` 备份恢复

#### 尝试3: 安全标签内文本替换（safe_i18n_v3.py）— 成功
- **根因分析**: 排查发现白屏有两个原因：
  1. `AIAgentPanel` 组件缺少 `const { t } = useI18n();` 声明，导致 `t` 未定义
  2. 前两次替换触碰了属性值（`placeholder="..."` → `placeholder={t('...')}`）和 JS 字符串，破坏了 Babel 编译
- **策略调整**: 严格只替换 `>中文文字</tag>` 模式的纯标签间文本，绝不碰属性值和 JS 字符串
- **执行步骤**:
  1. 给 `AIAgentPanel` 组件添加 `const { t } = useI18n();`
  2. 向3个语言块各添加约120个新翻译 key（`safe_i18n_v3.py` 内置）
  3. 只替换 `>中文</tag>` 模式的112处文本
- **结果**: 112处替换成功，6处未匹配（部分文本前后空格或换行不一致），页面正常加载
- **部署**: 成功部署，EN/简/繁 三语切换功能可用

### 替换覆盖范围
| 组件 | 替换数量 | 说明 |
|------|----------|------|
| Dashboard | ~40处 | KPI标签、Agent卡片标题/描述/按钮、图表标题、表格头/标签/时间 |
| Sustainability | ~30处 | 页面标题、KPI标签、图表标题、报告列表、重点洞察 |
| Green Building | ~15处 | 预评分、范畴概览、认证进度、提升建议 |

### 未替换的部分（约78处）
- 属性值内的中文（如 `placeholder="搜索设备编号..."`、`設定` 按钮）
- Chart.js 标签（如 `['HVAC', '照明', '電力', ...]`）
- 表格数据中的地点名（如 `3/F 辦公園區 A`）
- 混合中英文短文本（如 `+2.4% 較上月`、`每 30 秒`）
- Finance 页面（已是英文硬编码，未做 i18n）
- 绿色建筑子页面部分细节文本

### 核心经验
- **根因**: `AIAgentPanel` 缺少 `useI18n()` hook 是导致白屏的关键原因之一
- **安全规则**: 只替换 `>中文</tag>` 模式，不碰属性和 JS 字符串
- **Chart.js 限制**: Chart.js 标签在 `useEffect` 中初始化，无法直接使用 React 的 `t()` 函数，需要额外处理
- Babel 实时编译对 JSX 语法极其敏感，`{}` 在不同上下文有不同含义
