# ESG VISA

# 一、系统概述

ESG VISA 是一个基于 Hyperledger Fabric 区块链和 GoFrame 框架的数字身份认证与建筑数据管理系统。系统为可持续建筑验证提供数字身份管理、建筑数据记录、文件加密存储、碳排放追踪、ESG 分析报告、预测性维护、绿色金融等能力。前端采用单页应用架构，支持多语言，提供统一的导航架构和核心功能模块。

系统采用去中心化身份 (DID) 技术, 每栋建筑拥有唯一的身份标识和 Hash ID, 所有建筑数据存储在区块链上, 确保不可篡改和可追溯。文件上传采用分片加密存储, 支持将文件分片上传到 IPFS 网络, 并在区块链上记录文件元数据和交易哈希, 实现建筑数据的可信存储和验证。

# 二、前端架构与导航系统

# 2.1 统一导航架构

系统采用统一的侧边导航栏架构，为所有功能模块提供一致的导航体验。导航栏位于页面左侧，采用固定宽度设计，包含系统Logo、用户信息快捷入口和主要功能模块导航项。导航栏采用深色背景，与主内容区域形成视觉对比，突出导航的重要性。

导航栏顶部显示ESG VISA系统Logo和系统名称，点击Logo可以返回 Dashboard总览页面。Logo下方显示当前登录用户的头像、姓名和角色信息，点击用户信息区域可以快速访问用户详情页面。这种设计让用户随时了解自己的身份状态，并能够快速切换用户相关功

能。

主要功能模块以导航项形式展示，每个导航项包含图标、文字标签和可选的徽章提示。导航项采用分组设计，核心功能模块（ Dashboard、Smart Profile、Data Vault）位于顶部，业务功能模块（Carbon Transparency、ESG Analytics、Predictive Maintenance）位于中间，系统功能模块（Green Finance、Settings、Help）位于底部。这种分组设计帮助用户快速理解系统功能结构，提高导航效率。

# 2.2核心功能模块导航

Dashboard是系统的总览页面，作为用户登录后的默认首页。导航项使用仪表板图标，点击后跳转到Dashboard页面，显示用户所有建筑项目的概览信息。Smart Profile是建筑智能档案模块，导航项使用建筑图标，显示建筑的基本信息、ESG评级、碳排放数据等核心指标。

Data Vault 是数据保险库模块，导航项使用保险库图标，提供文件和数据加密存储功能。该模块支持 IPFS 存储，用户可以上传、下载、管理建筑相关的各类文件和数据。Carbon Transparency 是碳排放透明度模块，导航项使用碳足迹图标，提供碳排放追踪和数据核证功能。

ESG Analytics and Reporting 是 ESG 分析报告模块，导航项使用图表图标，提供 ESG 报告生成和分析功能。该模块与 AI 接口集成，能够自动生成 ESG 分析报告。Predictive Maintenance 是预测性维护模块，导航项使用维护图标，提供建筑维护预测和计划功能。

Green Finance and Insurance 是绿色金融与保险模块，导航项使用

金融图标，提供与金融机构和保险公司的数据接口。该模块允许授权合作伙伴访问特定的建筑数据，用于绿色金融产品设计和保险风险评估。

# 2.3 导航交互设计

导航栏采用响应式设计，在桌面端保持固定显示，在移动端可以折叠为汉堡菜单。当前激活的导航项会高亮显示，使用系统主色调的背景色和图标颜色，让用户清楚知道当前所在的功能模块。导航项支持悬停效果，鼠标悬停时会有背景色变化和轻微位移，提供良好的交互反馈。

导航栏支持搜索功能，用户可以在导航栏顶部输入关键词快速搜索功能模块。搜索结果会高亮显示匹配的导航项，帮助用户快速定位目标功能。这种设计特别适合功能模块较多的场景，提高导航效率。

系统还提供了面包屑导航，在页面顶部显示当前页面的路径，如" Dashboard > Smart Profile > 建筑名称"。用户可以点击面包屑中的任意层级快速返回上级页面，这种设计增强了导航的灵活性和用户体验。

# 三、 Dashboard 仪表板总览

# 3.1总览页面布局

Dashboard 是用户登录后的默认首页，提供系统所有关键信息的概览视图。页面采用网格布局，顶部显示欢迎信息和用户基本信息，包括用户姓名、角色、登录时间等。下方分为多个卡片区域，每个卡片展示不同类型的信息。

页面顶部左侧显示用户当前管理的建筑项目数量、总碳排放量、总体ESG评级等核心指标。这些指标以大号数字和图标形式展示，让用户一眼就能了解整体情况。右侧显示快速操作按钮，包括"添加新建筑"、"上传文件"、"生成报告"等常用功能，方便用户快速执行操作。

# 3.2 建筑项目列表

Dashboard 中央区域显示用户管理的所有建筑项目列表。每个建筑项目以卡片形式展示，卡片包含建筑缩略图、建筑名称、建筑地址、唯一 Hash ID、ESG 评级、碳排放统计、文件上传状态等关键信息。卡片采用悬停效果，鼠标悬停时会显示更多操作按钮，如"查看详情"、"编辑信息"、"上传数据"等。

建筑项目卡片按照ESG评级或最近更新时间排序，用户可以通过筛选器选择排序方式。每个卡片顶部显示建筑的唯一Hash ID，这是一个基于区块链生成的唯一标识符，用于在区块链上查询和验证建筑信息。Hash ID采用短格式显示，点击可以查看完整的Hash值。

卡片中央显示建筑的ESG评级，使用星级或字母等级（如 $\mathrm{A}+$ 、A、B+等）展示。评级旁边显示评级更新时间，让用户了解数据的时效性。卡片底部显示碳排放统计，包括总碳排放量、能源消耗、水消耗等关键指标，以图表或数字形式展示。

# 3.3 ESG指标总览

Dashboard 右侧显示 ESG 指标总览面板，以图表形式展示所有建筑项目的 ESG 表现。面板包含多个图表，包括 ESG 评级分布饼图、碳排放趋势折线图、能源消耗对比柱状图等。这些图表帮助用户快速了

解整体ESG表现和趋势。

ESG指标面板支持时间范围筛选，用户可以选择查看最近一周、一个月、一个季度或一年的数据。面板还支持建筑项目筛选，用户可以选择查看特定建筑或所有建筑的数据。这种灵活的筛选功能帮助用户从不同角度分析ESG数据。

面板底部显示关键指标汇总，包括平均ESG评级、总碳排放量、总能源消耗、总水消耗等。这些汇总数据帮助用户快速了解整体情况，并与历史数据进行对比，识别改进趋势。

# 3.4 文件上传状态监控

Dashboard 底部显示文件上传状态监控面板，实时显示最近的文件上传活动。面板以时间线形式展示文件上传记录，包括文件名、上传时间、上传状态（成功、失败、进行中）、文件类型等信息。用户可以点击任意记录查看文件详情或重新上传失败的文件。

面板还显示文件存储统计信息，包括已存储文件总数、总存储容量、IPFS节点数量等。这些信息帮助用户了解数据存储情况，确保重要数据得到妥善保存。面板提供"查看所有文件"链接，点击后跳转到Data Vault模块，查看完整的文件列表。

# 四、Smart Profile 建筑智能档案

# 4.1 建筑基本信息展示

Smart Profile 是建筑智能档案模块，为每栋建筑提供完整的数据档案页面。页面顶部显示建筑的基本信息卡片，包括建筑名称、建筑地址、唯一 Hash ID、建筑类型、建筑面积、建筑年代等。这些信息以清

晰的标签和值的形式展示，便于用户快速了解建筑的基本情况。

建筑地址信息支持地图显示，用户可以点击地址查看建筑在地图上的位置。地图集成支持多种地图服务提供商，包括高德地图、百度地图、Google Maps等，根据用户所在地区自动选择合适的地图服务。地图上会标注建筑位置，并显示周边环境信息，帮助用户了解建筑的地理环境。

唯一HashID是建筑在区块链上的唯一标识符，采用Base58编码格式显示，便于复制和分享。HashID旁边提供"复制"按钮和"查看区块链记录"链接，用户可以快速复制HashID或查看该建筑在区块链上的完整记录。这种设计增强了建筑身份的可验证性和透明度。

# 4.2 碳排放数据展示

Smart Profile 页面中央区域显示建筑的碳排放数据，这是 ESG 评估的核心指标之一。碳排放数据以多个维度展示，包括总碳排放量（Total Carbon Emission）、年度碳排放趋势、碳排放来源分布、碳排放强度等。总碳排放量以大号数字显示，单位为吨二氧化碳当量（tCO2e），旁边显示与行业平均值的对比。

碳排放趋势图以折线图形式展示建筑在过去一年或更长时间内的碳排放变化趋势。图表支持时间范围选择，用户可以选择查看月度、季度或年度数据。趋势图帮助用户识别碳排放的变化模式，识别季节性变化或长期趋势，为减排措施提供数据支持。

碳排放来源分布以饼图或堆叠柱状图形式展示，显示不同来源的碳排放占比，如能源消耗、建筑材料、交通运输、废弃物处理等。这种

分解帮助用户了解碳排放的主要来源，有针对性地制定减排策略。每个来源旁边显示具体的碳排放量和占比，便于用户进行详细分析。

碳排放强度指标显示单位面积或单位产值的碳排放量，这是评估建筑碳排放效率的重要指标。系统会计算建筑的碳排放强度，并与同类建筑的平均值进行对比，显示建筑的碳排放效率水平。这种对比帮助用户了解建筑在行业中的表现，识别改进空间。

# 4.3 能源效率数据展示

Smart Profile页面显示建筑的能源效率数据，包括能源消耗总量、能源节约率（Energy Saving Rate）、能源消耗趋势、能源来源分布等。能源节约率是衡量建筑能源效率的关键指标，显示建筑相比基准建筑的能源节约百分比。该指标以大号数字和百分比形式展示，旁边显示与行业平均值的对比。

能源消耗趋势图以折线图形式展示建筑在过去一段时间内的能源消耗变化，包括电力消耗、天然气消耗、其他能源消耗等。图表支持多能源类型叠加显示，帮助用户全面了解能源消耗情况。系统还会计算能源消耗的季节性变化，识别能源消耗的高峰期和低谷期。

能源来源分布图显示建筑使用的不同能源类型的占比，如电力、天然气、太阳能、风能等。对于使用可再生能源的建筑，系统会特别标注可再生能源的占比，这是ESG评估的重要加分项。系统还会计算可再生能源的发电量或使用量，显示建筑在清洁能源使用方面的表现。

能源效率评级显示建筑的能源效率等级，通常分为 $\mathrm{A}+$ 、A、B+、B、C等等级。评级基于建筑的能源消耗强度、能源节约率、

可再生能源使用比例等多个因素综合计算。系统会显示评级的计算依据和各项指标的得分，帮助用户了解评级的构成。

# 4.4 维护状态展示

Smart Profile 页面显示建筑的维护状态（Maintenance Status），这是评估建筑可持续性和运营效率的重要指标。维护状态以状态卡片形式展示，包括当前维护状态（良好、一般、需要维护、紧急维护）、最近维护时间、下次维护计划、维护历史记录等。

当前维护状态使用颜色编码显示，绿色表示状态良好，黄色表示需要注意，红色表示需要紧急维护。状态旁边显示状态更新时间，让用户了解数据的时效性。系统还会显示维护状态的趋势，帮助用户识别维护状态的变化模式。

维护历史记录以时间线形式展示，显示建筑过去的所有维护活动，包括维护类型、维护时间、维护内容、维护成本等。用户可以点击任意维护记录查看详细信息，了解维护的具体内容和效果。系统还会分析维护频率和维护成本，帮助用户优化维护计划。

下次维护计划显示系统根据建筑使用情况和维护历史预测的下次维护时间和维护内容。对于启用了 IoT 数据采集的建筑，系统会结合 IoT 设备的实时数据，提供更准确的维护预测。用户可以查看维护计划的详细内容，并根据实际情况调整维护计划。

# 4.5 ESG评级与报告

Smart Profile 页面顶部显著位置显示建筑的 ESG 评级（ESG Rating），这是综合评估建筑在环境、社会、治理三个维度表现的指标。

ESG 评级通常以字母等级形式展示，如 A+、A、B+、B、C 等，或者以星级形式展示，如 5 星、4 星等。评级旁边显示评级的更新时间，让用户了解数据的时效性。

ESG评级基于多个维度的数据综合计算，包括碳排放、能源效率、水资源使用、材料使用、社会责任、治理结构等。系统会显示每个维度的得分和权重，帮助用户了解评级的构成。用户可以点击评级查看详细的评级报告，了解各项指标的得分和改进建议。

ESG Report（ESG报告）是建筑的完整ESG评估报告，包含详细的数据分析、趋势预测、改进建议等内容。报告以PDF或网页形式展示，用户可以下载或在线查看。报告包含多个章节，包括执行摘要、环境表现、社会表现、治理表现、数据核证、改进建议等。

报告的数据来源包括用户手动输入的数据、IoT设备采集的数据、第三方数据源等。系统会标注每个数据项的数据来源和更新时间，确保报告的透明度和可信度。报告还包含数据核证信息，显示哪些数据已经过第三方核证，哪些数据是系统自动计算的。

# 4.6 数据获取方式

Smart Profile 的数据获取采用多种方式，确保数据的完整性和准确性。首先，用户可以通过手动输入方式录入建筑的基本信息、维护记录、能源消耗等数据。系统提供友好的数据录入界面，支持批量导入和单个录入，并包含数据验证功能，确保录入数据的准确性。

其次，系统支持 IoT 设备数据采集，对于安装了智能传感器的建筑，系统可以自动采集能源消耗、室内环境、设备运行状态等实时数

据。IoT数据通过标准协议（如MQTT、HTTP）传输到系统，系统会进行数据清洗和验证，确保数据的质量。IoT数据采集大大提高了数据的实时性和准确性，减少了人工录入的工作量。

第三，系统支持第三方数据源集成，如能源供应商的数据、政府公开数据、认证机构的数据等。系统通过API接口或数据导入方式获取第三方数据，并与建筑数据关联，丰富数据内容。第三方数据的集成需要经过用户授权，确保数据使用的合规性。

第四，系统支持数据核证功能，用户可以邀请第三方认证机构对建筑数据进行核证。核证机构通过系统提供的接口上传核证报告和核证数据，系统会将核证信息与原始数据关联，在Smart Profile中显示核证状态。核证数据具有更高的可信度，可以用于ESG评级和报告生成。

# 五、Data Vault 数据保险库

# 5.1 数据保险库界面设计

Data Vault 是系统的数据保险库模块，提供文件和数据加密存储功能。新版本的 Data Vault 界面采用统一的导航架构，与系统其他模块保持一致的视觉风格。页面左侧是侧边导航栏，右侧是主内容区域，顶部显示模块标题和用户信息。

主内容区域分为多个功能区，包括文件上传区、文件列表区、存储统计区等。文件上传区采用拖拽上传设计，用户可以将文件拖拽到指定区域，或点击按钮选择文件。上传区域显示支持的文件类型、最大文件大小、加密说明等信息，帮助用户了解上传要求。

文件列表区以表格或卡片形式展示所有已上传的文件，每个文件项显示文件名、文件类型、文件大小、上传时间、加密状态、IPFS链接、区块链哈希等信息。文件列表支持搜索、筛选、排序功能，用户可以快速找到目标文件。列表还支持批量操作，如批量下载、批量删除等。

# 5.2加密存储功能

Data Vault 模块的核心功能是文件加密存储，所有上传的文件都会经过加密处理后再存储到 IPFS 网络。加密过程在用户浏览器中完成，使用 AES-256 加密算法，确保文件内容在传输和存储过程中始终处于加密状态。系统会为每个文件生成唯一的加密密钥，密钥的哈希值存储在区块链上，用于验证密钥的完整性。

文件上传时，系统会将大文件分割成多个固定大小的分片（通常为2MB或5MB），每个分片独立加密后上传到IPFS网络。这种分片设计具有多个优势：首先，提高了上传的可靠性，即使部分分片上传失败，其他分片仍然可以成功上传；其次，提高了下载的并行性，可以同时下载多个分片，加快下载速度；最后，提高了存储的灵活性，可以根据需要选择性地下载部分分片。

文件元数据（包括文件名、描述、上传者、分片信息、加密密钥哈希、IPFS链接等）会记录在区块链上，通过HyperledgerFabric的链码实现。区块链记录确保文件元数据的不可篡改性，任何人都可以通过交易哈希验证文件的真实性。系统还会为每个文件生成唯一的文件ID，用于在系统中标识和检索文件。

# 5.3 IPFS 存储集成

Data Vault 模块深度集成 IPFS 网络，所有加密文件分片都存储在 IPFS 网络上。IPFS 是一个分布式文件系统，文件存储在 IPFS 网络的多个节点上，具有高可用性和去中心化特性。系统会为每个文件生成 IPFS Content ID（CID），这是文件在 IPFS 网络上的唯一标识符。

文件上传到IPFS后，系统会显示文件的IPFS链接，用户可以通过该链接在其他支持IPFS的应用中访问文件。IPFS链接采用标准格式，如`ipfs://QmXXX...`或通过IPFS网关访问，如`https://ipfs.io/ipfs/QmXXX...`。系统还提供IPFS节点状态监控，显示当前连接的IPFS节点数量、存储容量、网络状态等信息。

系统支持IPFS文件的固定（Pin）功能，确保重要文件不会被IPFS网络的垃圾回收机制删除。用户可以标记重要文件为"固定"，系统会将这些文件固定到多个IPFS节点上，确保文件的长期可用性。固定功能对于重要的建筑数据文件特别有用，确保这些文件不会丢失。

# 5.4 文件管理功能

Data Vault 模块提供完整的文件管理功能，用户可以上传、下载、查看、删除文件。文件上传支持单个文件和批量上传，支持拖拽上传和点击选择上传。上传过程中，系统会显示上传进度，包括已上传的分片数量、上传速度、剩余时间等信息。上传完成后，系统会显示文件的上传结果，包括 IPFS 链接、区块链交易哈希等。

文件下载支持普通下载和加密文件下载。对于加密文件，系统会自动完成下载、解密、合并等操作，用户最终得到解密后的完整文件。  
下载过程中，系统会显示详细的下载进度，包括下载速度、剩余时间、

已下载大小等信息。用户还可以选择只下载部分分片，适用于大文件的预览场景。

文件查看功能支持在线预览，对于图片、PDF、文本等文件类型，系统可以在浏览器中直接预览，无需下载。预览功能使用IPFS网关或系统内置的预览服务，确保预览的安全性和速度。对于不支持预览的文件类型，系统会显示文件的基本信息和下载按钮。

文件删除功能支持单个删除和批量删除。删除操作会从IPFS网络和区块链上删除文件记录，但需要注意的是，IPFS网络上的文件可能仍然存在于其他节点上，直到被垃圾回收。系统会提示用户删除操作的不可逆性，并要求用户确认删除操作。

# 5.5存储统计与监控

Data Vault 模块提供存储统计和监控功能，帮助用户了解数据存储情况。存储统计面板显示已存储文件总数、总存储容量、已用存储空间、可用存储空间等信息。统计信息以图表形式展示，包括存储容量趋势图、文件类型分布图、存储位置分布图等。

系统还会显示每个文件的存储状态，包括IPFS节点数量、文件可用性、最后访问时间等。对于重要文件，系统会特别标注，并显示文件的备份状态。用户可以通过存储统计面板了解数据存储的健康状况，及时发现和解决存储问题。

系统还提供存储优化建议，如清理重复文件、压缩大文件、归档旧文件等。这些建议帮助用户优化存储空间，降低存储成本。用户可以按照建议执行优化操作，系统会提供操作向导和进度反馈。

# 六、Carbon Transparency 碳排放透明度

# 6.1 碳排放追踪界面

Carbon Transparency 模块提供碳排放追踪和数据核证功能，帮助用户全面了解建筑的碳排放情况。页面采用数据可视化设计，顶部显示建筑的总体碳排放量，以大号数字和趋势箭头展示，让用户一眼就能了解碳排放的总体情况。

页面中央区域显示碳排放的详细追踪信息，包括碳排放时间线、碳排放来源分解、碳排放强度分析等。碳排放时间线以时间轴形式展示建筑在过去一段时间内的碳排放记录，每个时间点显示该时间段的碳排放量、主要排放来源、与基准值的对比等信息。时间线支持缩放和筛选，用户可以查看不同时间范围的数据。

碳排放来源分解以饼图或堆叠柱状图形式展示，显示不同来源的碳排放占比，如能源消耗、建筑材料、交通运输、废弃物处理、水资源使用等。每个来源旁边显示具体的碳排放量、占比、与行业平均值的对比等信息。用户可以点击任意来源查看该来源的详细数据和分析。

# 6.2 数据核证功能

Carbon Transparency 模块提供数据核证功能，确保碳排放数据的准确性和可信度。数据核证包括自动核证和第三方核证两种方式。自动核证由系统根据数据来源、数据质量、数据一致性等因素自动进行，核证结果以标签形式显示在数据项旁边。

第三方核证由认证机构或审计机构进行，核证机构通过系统提供的接口上传核证报告和核证数据。系统会将核证信息与原始数据关联，

在 Carbon Transparency 页面中显示核证状态。核证数据具有更高的可信度，可以用于 ESG 评级和报告生成。

数据核证状态以颜色编码显示，绿色表示已核证，黄色表示待核证，红色表示核证失败。用户可以点击核证状态查看核证的详细信息，包括核证机构、核证时间、核证方法、核证结果等。系统还会显示核证数据的有效期，提醒用户及时更新核证数据。

# 6.3 碳排放分析

Carbon Transparency 模块提供碳排放分析功能，基于 Smart Profile 中的数据进行分析和可视化。分析功能包括碳排放趋势分析、碳排放对比分析、碳排放预测分析等。碳排放趋势分析显示建筑碳排放的历史变化趋势，识别季节性变化、长期趋势、异常波动等模式。

碳排放对比分析将建筑的碳排放与同类建筑、行业平均值、基准值等进行对比，帮助用户了解建筑在行业中的表现。对比分析以图表形式展示，包括柱状图等，用户可以直观地看到建筑的优劣势和改进空间。

碳排放预测分析基于历史数据和机器学习算法，预测建筑未来的碳排放趋势。预测分析考虑多个因素，如建筑使用情况、能源消耗模式、维护计划、改造计划等。预测结果以折线图形式展示，显示预测的碳排放量和置信区间，帮助用户制定减排策略。

# 七、ESG Analytics and Reporting ESG 分析报告

# 7.1 报告生成界面

ESG Analytics and Reporting 模块提供 ESG 报告生成和分析功能，

与AI接口集成，能够自动生成ESG分析报告。页面顶部显示报告生成区域，用户可以选择报告类型(年度报告、季度报告、专项报告等)、报告时间范围、报告模板等。选择完成后，点击"生成报告"按钮，系统会调用AI接口生成报告。

报告生成过程中，系统会显示生成进度，包括数据收集、数据分析、报告撰写、报告格式化等步骤。AI接口会分析建筑的ESG数据，包括碳排放、能源效率、维护状态、社会责任、治理结构等多个维度，生成综合性的ESG分析报告。

生成的报告以网页形式展示，包含多个章节，包括执行摘要、环境表现分析、社会表现分析、治理表现分析、数据核证、改进建议、未来展望等。报告采用专业的排版和设计，包含图表、数据表格、文字分析等内容，确保报告的专业性和可读性。

# 7.2 AI分析结果展示

ESG Analytics and Reporting 模块与 AI 接口联调，获取 AI 对建筑 ESG 数据的分析结果。AI 分析结果以多个维度展示，包括 ESG 评级分析、趋势预测分析、对比分析、改进建议分析等。每个分析结果以卡片形式展示，包含分析标题、分析内容、数据支撑、置信度等信息。

ESG评级分析显示AI对建筑ESG评级的评估结果，包括总体评级、各维度评级、评级依据、评级趋势等。AI会基于历史数据和行业标准，评估建筑的ESG表现，并提供详细的评级说明。评级分析还包含与同类建筑的对比，帮助用户了解建筑在行业中的位置。

趋势预测分析显示AI对建筑未来ESG表现的预测，基于历史数

据、行业趋势、政策变化等因素进行预测。预测结果以图表形式展示，显示预测的ESG评级、碳排放趋势、能源效率趋势等。预测分析还包含不确定性分析，显示预测的置信区间和可能的变化范围。

对比分析将建筑的ESG表现与同类建筑、行业平均值、最佳实践等进行对比，识别建筑的优劣势和改进机会。对比分析以雷达图、柱状图、散点图等形式展示，用户可以直观地看到建筑的相对表现。AI还会提供针对性的改进建议，帮助用户提升ESG表现。

# 7.3 报告定制与导出

ESG Analytics and Reporting 模块支持报告定制功能，用户可以选择报告包含的内容、报告格式、报告样式等。报告内容可以选择包含哪些章节、哪些数据项、哪些图表等，用户可以根据需要定制报告的内容和结构。报告格式支持 PDF、Word、Excel、网页等多种格式，用户可以根据使用场景选择合适的格式。

报告导出功能支持一键导出，用户点击"导出报告"按钮，系统会生成报告文件并提供下载链接。导出的报告包含完整的内容和格式，可以直接用于提交、分享、存档等用途。系统还会保存报告的历史版本，用户可以查看和下载之前的报告版本。

报告分享功能支持生成分享链接，用户可以将报告链接分享给其他用户或机构。分享链接可以设置访问权限，如公开访问、需要密码、仅限特定用户等。系统还会记录报告的访问记录，用户可以查看报告的访问情况。

# 八、Green Finance and Insurance 绿色金融与保险

# 8.1 数据接口设计

Green Finance and Insurance 模块提供与金融机构和保险公司的数据接口，允许授权合作伙伴访问特定的建筑数据，用于绿色金融产品设计和保险风险评估。页面顶部显示接口配置区域，用户可以设置哪些数据可以共享、共享给哪些合作伙伴、共享的数据范围等。

数据接口采用RESTful API设计，支持标准的数据交换格式，如JSON、XML等。接口提供建筑的基本信息、ESG数据、碳排放数据、能源效率数据、维护状态数据等。合作伙伴可以通过API接口获取这些数据，用于产品设计和风险评估。

接口访问需要身份认证和授权，合作伙伴需要先注册账户，获得API密钥，然后使用密钥访问接口。系统会记录所有接口访问记录，包括访问时间、访问数据、访问结果等，确保数据使用的可追溯性。

# 8.2 数据共享配置

Green Finance and Insurance 模块提供数据共享配置功能，用户可以精确控制哪些数据可以共享给合作伙伴。数据共享配置包括共享数据范围、共享数据格式、共享数据更新频率等。用户可以设置共享建筑的基本信息、ESG 评级、碳排放数据、能源效率数据等，也可以选择不共享某些敏感数据。

数据共享支持多种方式，包括实时数据推送、定期数据导出、按需数据查询等。实时数据推送适用于需要实时数据的场景，如实时风险评估、实时产品定价等。定期数据导出适用于需要批量数据的场景，如定期报告、定期分析等。按需数据查询适用于临时数据需求，如临

时评估、临时分析等。

系统还提供数据共享协议管理功能，用户可以与合作伙伴签署数据共享协议，明确数据使用的范围、目的、期限等。协议信息会记录在系统中，系统会根据协议自动控制数据共享的权限和范围。

# 8.3 合作伙伴管理

Green Finance and Insurance 模块提供合作伙伴管理功能，用户可以管理所有授权访问数据的合作伙伴。合作伙伴列表显示所有合作伙伴的信息，包括合作伙伴名称、合作伙伴类型（金融机构、保险公司、评估机构等）、授权状态、访问权限、最后访问时间等。

用户可以添加、编辑、删除合作伙伴，设置合作伙伴的访问权限和数据范围。系统会为每个合作伙伴生成唯一的API密钥，合作伙伴使用该密钥访问数据接口。系统还会记录合作伙伴的所有访问记录，用户可以查看合作伙伴的数据使用情况。

系统还提供合作伙伴数据分析功能，显示哪些合作伙伴访问了哪些数据、访问频率、数据使用目的等。这些分析帮助用户了解数据共享的情况，优化数据共享策略。

# 8.4 用户身份与角色管理

系统建立了完善的用户身份与角色管理模块，支持未来权限分级设计。角色分类包括系统管理员、建筑管理员、数据录入员、查看者、合作伙伴等。每个角色都有明确的权限范围，系统管理员拥有最高权限，可以管理所有用户和建筑；建筑管理员可以管理自己负责的建筑；数据录入员可以录入和编辑数据；查看者只能查看数据；合作伙伴只

能访问授权的数据。

角色管理功能包括角色创建、角色编辑、角色删除、角色权限配置等。系统管理员可以创建新角色，设置角色的权限范围，然后将角色分配给用户。角色权限配置支持细粒度的权限控制，可以精确控制每个角色可以访问哪些功能、哪些数据、执行哪些操作。

用户身份管理包括用户注册、用户认证、用户信息管理、用户状态管理等。用户注册需要提供基本信息并通过邮箱验证，注册成功后系统会生成唯一的 DID标识符。用户认证支持多种方式，包括DID登录、邮箱密码登录、微信登录等。用户信息管理允许用户查看和编辑自己的信息，系统管理员可以管理所有用户的信息。用户状态管理包括用户激活、用户禁用、用户删除等操作。

权限分级设计支持多级权限体系，如系统级权限、建筑级权限、数据级权限等。系统级权限控制用户对系统功能的访问，建筑级权限控制用户对特定建筑的访问，数据级权限控制用户对特定数据的访问。这种多级权限设计确保了系统的安全性和灵活性，可以满足不同场景的权限需求。
