# 多语言转换功能实现总结

## 实现方案

根据master的要求，实现了以下多语言转换方案：

### 核心设计原则
- **只有index界面有语言转换按键** - 用户只能在首页切换语言
- **其他界面自动跟随** - 所有其他页面根据index页面的语言选择自动翻译
- **统一语言配置** - 所有页面共享同一套语言数据和翻译逻辑

## 文件结构

### 1. 核心配置文件
- `language-config.js` - 统一语言配置和翻译逻辑
- `auto-translate.js` - 自动翻译脚本（用于非index页面）

### 2. 页面修改
- `index.html` - 保留语言转换按键，作为语言控制中心
- `register1.html` - 移除语言转换按键，添加自动翻译
- `register12.html` - 移除语言转换按键，添加自动翻译  
- `register-2.html` - 移除语言转换按键，添加自动翻译
- `login1.html` - 微信扫码登录页面，风格与register-2.html一致，支持多语言转换

## 技术实现

### 1. 语言数据配置
```javascript
const languageData = {
    en: { code: 'EN', flag: '🇺🇸', name: 'English', translations: {...} },
    'zh-cn': { code: '简体', flag: '🇨🇳', name: '简体中文', translations: {...} },
    'zh-tw': { code: '繁體', flag: '🇭🇰', name: '繁體中文', translations: {...} }
};
```

### 2. 翻译属性系统
使用 `data-translate` 属性标记需要翻译的元素：
```html
<h1 data-translate="verification.title">Verification required</h1>
<input placeholder="Enter your email" data-translate="email.placeholder">
<button data-translate="verification.verify">Verify</button>
```

### 3. 自动翻译机制
- 页面加载时自动应用当前语言设置
- 监听localStorage变化，实现跨页面语言同步
- 支持输入框placeholder、按钮文本、标题等元素翻译

## 使用方法

### 1. 在index页面切换语言
用户点击右上角的语言选择器，选择目标语言：
- 点击"EN"选择英语
- 点击"简体"选择简体中文  
- 点击"繁體"选择繁体中文

### 2. 其他页面自动跟随
- 所有其他页面会自动应用index页面选择的语言
- 无需在每个页面重复设置语言
- 语言设置保存在localStorage中，刷新页面后保持

## 维护说明

### 1. 添加新翻译
在 `language-config.js` 的 `translations` 对象中添加新的键值对：
```javascript
translations: {
    'new.key': 'New Translation Text'
}
```

### 2. 添加新语言
在 `languageData` 对象中添加新的语言配置：
```javascript
'new-lang': {
    code: 'NEW',
    flag: '🏳️',
    name: 'New Language',
    translations: { ... }
}
```

### 3. 添加新页面
在新页面中：
1. 添加 `data-translate` 属性到需要翻译的元素
2. 引入 `language-config.js` 和 `auto-translate.js`
3. 确保页面有正确的 `data-ui-id` 属性
