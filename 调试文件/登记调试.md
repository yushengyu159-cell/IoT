## 一、Google登录功能实现

### 1.1 实现内容
- 在 `index.html` 中为Google登录按钮添加事件监听器
- 点击Google按钮时检查邮箱是否存在：
  - 如果邮箱存在：跳转到 `register1.html#login`（登录界面）
  - 如果邮箱不存在：跳转到 `register1.html#register`（注册界面）
- 与"继续"按钮行为保持一致

### 1.2 代码修改
```javascript
// index.html 中添加
document.querySelector('.social-google').addEventListener('click', function() {
    const email = document.getElementById('email').value.trim();
    if (!email) {
        alert('请输入邮箱地址');
        return;
    }
    
    // 检查邮箱是否存在
    fetch('/api/register/check-email', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ email: email })
    })
    .then(response => response.json())
    .then(data => {
        if (data.code === 200 && data.data.exists) {
            window.location.href = '/static/register1.html#login';
        } else {
            window.location.href = '/static/register1.html#register';
        }
    })
    .catch(error => {
        console.error('检查邮箱失败:', error);
        window.location.href = '/static/register1.html#register';
    });
});
```

### 1.3 测试结果
- ✅ Google登录按钮功能正常
- ✅ 邮箱存在时正确跳转到登录界面
- ✅ 邮箱不存在时正确跳转到注册界面

## 二、WeChat登录功能实现

### 2.1 实现内容
- 在 `index.html` 中为WeChat登录按钮添加事件监听器
- 点击WeChat按钮时检查邮箱是否存在：
  - 如果邮箱存在：跳转到 `login1.html`（微信二维码登录界面）
  - 如果邮箱不存在：跳转到 `register1.html#register`（注册界面）

### 2.2 后端API实现

#### 2.2.1 新增控制器文件
- 文件：`/root/home/go/fabric-sdk/internal/controller/wechat.go`
- 实现4个API接口：
  1. `POST /api/wechat/check-and-generate-qrcode` - 检查邮箱并生成二维码
  2. `GET /api/wechat/check-login-status` - 检查登录状态
  3. `POST /api/wechat/confirm-login` - 确认登录
  4. `POST /api/wechat/scan-qrcode` - 扫码状态更新

#### 2.2.2 路由配置
- 在 `cmd.go` 中添加WeChat相关路由：
```go
s.Group("/api/wechat", func(group *ghttp.RouterGroup) {
    group.POST("/check-and-generate-qrcode", controller.Wechat.CheckAndGenerateQRCode)
    group.GET("/check-login-status", controller.Wechat.CheckLoginStatus)
    group.POST("/confirm-login", controller.Wechat.ConfirmWeChatLogin)
    group.POST("/scan-qrcode", controller.Wechat.ScanQRCode)
})
```

#### 2.2.3 缓存机制
- 使用 `gcache` 内存缓存存储登录状态
- Token有效期：30分钟
- 状态流转：`waiting` → `scanned` → `confirmed`

### 2.3 前端页面实现

#### 2.3.1 login1.html（PC端二维码页面）
- 显示二维码
- 轮询检查登录状态（每3秒）
- 状态显示：
  - `waiting`: 请使用微信扫描二维码
  - `scanned`: 二维码已扫描，请在手机上确认登录
  - `confirmed`: 登录成功，跳转到文件管理界面
  - `expired`: 二维码已过期

#### 2.3.2 wechat-confirm.html（手机端确认页面）
- 显示确认登录界面
- 提供"确认登录"和"取消"按钮
- 确认后显示"登录成功！请在电脑上查看"
- 尝试关闭窗口或提示用户关闭标签页

### 2.4 二维码生成
- 使用 `github.com/skip2/go-qrcode` 库
- 二维码内容：`http://47.238.159.234:8199/static/wechat-confirm.html?token={uuid}`
- 使用 `github.com/google/uuid` 生成唯一token
