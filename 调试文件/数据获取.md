# 数据获取失败问题调试记录

## 问题描述
用户界面显示"加载失败"和"获取邮箱列表失败"的错误信息，用户列表无法正常显示数据。

## 问题现象
- **前端表现**: 用户列表页面显示加载失败，提示"Loading users...获取邮箱列表失败"
- **API响应**: `/api/register/emails` 返回500错误
- **错误信息**: "获取邮箱列表失败"

## 问题分析过程

### 1. 初步检查
```bash
curl -s "http://localhost:8199/api/register/emails" | jq .
```
**结果**: 
```json
{
  "code": 500,
  "message": "获取邮箱列表失败",
  "data": null
}
```

### 2. 日志分析
```bash
docker logs fabric-sdk-app | tail -20
```
**发现关键错误**:
```
2025-10-20T14:36:33.159+08:00 [ERRO] SELECT `email` FROM `dids` WHERE status = 'completed' ORDER BY `created_at` DESC
Error: dial tcp 127.0.0.1:3306: connect: connection refused
```

### 3. 根本原因分析
**问题根源**: `GetAllRegisteredEmails` 方法使用了错误的数据库连接
- **错误方式**: 使用 `g.DB()` (GoFrame默认数据库连接)
- **正确方式**: 应该使用 `service.DB` (GORM数据库连接)

## 问题原因详解

### 1. 数据库连接配置不一致
- **应用启动时**: 使用 `service.DB` 连接MySQL (通过 `MYSQL_DSN` 环境变量)
- **API查询时**: 使用 `g.DB()` 连接 (使用配置文件中的连接字符串)

### 2. 连接字符串差异
- **service.DB**: `root:Test@123456@tcp(fabric-sdk-mysql:3306)/esg`
- **g.DB()**: `mysql:root:Test@123456@tcp(fabric-sdk-mysql:3306)/esg`

### 3. 网络连接问题
- `g.DB()` 尝试连接 `127.0.0.1:3306` (容器内部localhost)
- 但MySQL服务在 `fabric-sdk-mysql:3306` (Docker网络中的服务名)

## 解决方案

### 1. 修复数据库连接
**修改前**:
```go
// 从数据库获取所有注册成功的邮箱
db := g.DB()

// 只查询完全注册成功的用户邮箱（状态为completed）
result, err := db.Model("dids").Fields("email").Where("status = ?", "completed").Order("created_at DESC").All()
```

**修改后**:
```go
// 从数据库获取所有注册成功的邮箱
if DB == nil {
    return nil, fmt.Errorf("数据库连接未初始化")
}

// 只查询完全注册成功的用户邮箱（状态为completed）
var results []model.DID
err := DB.Where("status = ?", "completed").Order("created_at DESC").Find(&results).Error
```

### 2. 更新查询方式
- **从GoFrame查询**: `db.Model("dids").Fields("email").Where(...).All()`
- **改为GORM查询**: `DB.Where(...).Find(&results)`

### 3. 修复字段映射
**修改前**:
```go
for _, record := range result {
    email := record["email"].String()
    if email != "" {
        emails = append(emails, email)
    }
}
```

**修改后**:
```go
for _, record := range results {
    if record.Email != "" {
        emails = append(emails, record.Email)
    }
}
```

## 技术要点总结

### 1. 数据库连接一致性
- **问题**: 应用中使用了两套不同的数据库连接方式
- **解决**: 统一使用 `service.DB` (GORM连接)

### 2. Docker网络配置
- **问题**: 容器内应用尝试连接localhost而不是Docker服务名
- **解决**: 确保所有数据库连接都使用Docker服务名

### 3. 查询方式统一
- **问题**: GoFrame和GORM查询方式混用
- **解决**: 统一使用GORM查询方式

### 4. 字段映射正确性
- **问题**: GoFrame查询结果和GORM模型字段访问方式不同
- **解决**: 使用正确的模型字段访问方式

## 预防措施

### 1. 代码规范

- 避免混用不同的ORM框架

### 2. 配置管理
